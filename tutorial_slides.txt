================================================================
  Network Diagram Editor - 入門チュートリアル
  対象: ネットワーク・インターネットを学び始めた方
================================================================

  このチュートリアルを読めば、
  「このアプリが何のためにあるか」と
  「基本的な使い方」の 8 割が分かります。


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 01                                                    ║
║  このアプリは「地図」を描くツールです                        ║
╚══════════════════════════════════════════════════════════════╝

  インターネットやサーバの世界では、
  たくさんのコンピュータがつながって動いています。

  でも「どのコンピュータが、どこにあって、どう通信するか」を
  頭の中だけで整理するのはとても大変です。

  そこで登場するのが「ネットワーク構成図」です。

  ネットワーク構成図 = コンピュータとその通信経路を描いた地図

  このアプリは、その地図を手軽に、
  見やすく描けるように作られたツールです。


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 02                                                    ║
║  まず「ネットワーク構成図」ってどんなもの？                  ║
╚══════════════════════════════════════════════════════════════╝

  たとえば、Webサイトを1つ動かすのに、
  こんなコンピュータたちが協力して働いています。

    ┌──────────────────────────────────────────────────────┐
    │  [ユーザー] → [ロードバランサ] → [アプリサーバ×2]   │
    │                                  ↓                   │
    │                             [DBサーバ]               │
    │  [監視サーバ] → 全部のサーバを見張る                 │
    └──────────────────────────────────────────────────────┘

  これを図にすると、
  「どこで何が起きているか」が一目でわかるようになります。

  ★ 構成図があると何がうれしいの？
    ・障害が起きたときに原因を探しやすい
    ・新しいメンバーに説明するときに話が速い
    ・設計のミスに事前に気づける
    ・お客様への説明資料として使える


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 03                                                    ║
║  画面を見てみよう                                            ║
╚══════════════════════════════════════════════════════════════╝

  アプリを開くと3つのエリアに分かれています。

  ┌─────────┬──────────────────────────┬──────────────┐
  │         │                          │              │
  │  ① 左  │       ② 真ん中          │   ③ 右      │
  │  パネル │       キャンバス         │   パネル     │
  │         │                          │              │
  │ ノードや │   ここに構成図を描く    │ 選んだものの │
  │ エリアを │                          │ 詳細を編集   │
  │ 追加する │                          │              │
  └─────────┴──────────────────────────┴──────────────┘

  ① 左パネル   : 新しいノード（コンピュータ）や
                 ネットワークエリアを追加するボタン群
  ② キャンバス : 実際に構成図を描く作業スペース
  ③ 右パネル  : 選択中のノードや接続線の属性を編集

  ※ パネルの境界線をドラッグすると幅を変えられます。


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 04                                                    ║
║  「ノード」って何？                                          ║
╚══════════════════════════════════════════════════════════════╝

  構成図に登場するコンピュータや機器のことを
  このアプリでは「ノード」と呼びます。

  ノードには7種類あります。それぞれ色とバッジで見分けられます。

  ┌────────────────┬──────┬──────────────────────────────────┐
  │ 種別           │ 記号 │ 役割のイメージ                   │
  ├────────────────┼──────┼──────────────────────────────────┤
  │ Server         │  SV  │ 汎用サーバ（Webサーバ、APサーバ） │
  │ Load Balancer  │  LB  │ 通信を振り分ける交通整理役       │
  │ Database       │  DB  │ データを保存・管理する           │
  │ Monitor        │  MN  │ 他のサーバを監視する             │
  │ User/Client    │  UC  │ エンドユーザー・端末             │
  │ Firewall       │  FW  │ 不正通信を遮断する守り番         │
  │ Router         │  RT  │ ネットワーク間の通り道           │
  └────────────────┴──────┴──────────────────────────────────┘

  ノードには次の情報を設定できます。
    ・表示名   : 画面上に大きく表示される名前 (例: app-01)
    ・ホスト名 : OS上の名前 (例: app-01.internal)
    ・IPアドレス: ネットワーク上の住所 (例: 10.0.1.11)
    ・役割     : このサーバが何をしているか (例: APサーバ)
    ・備考     : 自由なメモ欄


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 05                                                    ║
║  ノードを追加してみよう                                      ║
╚══════════════════════════════════════════════════════════════╝

  ▼ 方法その1：左パネルのボタンから追加

    1. 左パネルの [Server] や [Database] などのボタンをクリック
    2. キャンバスの中央にノードが追加されます
    3. ドラッグして好きな位置に移動しましょう

  ▼ 方法その2：キャンバスをダブルクリック

    1. 左パネルで追加したい種別のボタンを一度クリック
       (これが「現在選択中の種別」になります)
    2. キャンバスの空いている場所をダブルクリック
    3. クリックした場所にノードが追加されます

  ▼ 追加したノードの名前などを変えるには

    1. ノードをクリックする
    2. 右パネルに詳細が表示されます
    3. 表示名・ホスト名・IPアドレスなどを入力してください

  ★ コツ: まず大まかにノードを並べてから名前をつけると
           全体のバランスが取りやすいです。


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 06                                                    ║
║  「ネットワークエリア」って何？                              ║
╚══════════════════════════════════════════════════════════════╝

  実際のネットワークでは、コンピュータは
  「セグメント」や「VLAN」と呼ばれるグループに分けられています。

  例えば…
    MGMT エリア : 管理用ネットワーク (踏み台・監視サーバ)
    DMZ  エリア : 外部に公開するネットワーク (LB・Webサーバ)
    APP  エリア : アプリケーションサーバ用
    DB   エリア : データベース用 (外部からは直接触れない)

  このアプリでは、こうしたセグメントを
  「色のついた背景の四角形」として表現します。

  ┌─────── MGMT (水色) ────────┐
  │  [jump-01]   [mon-01]       │
  └────────────────────────────┘
  ┌─── DMZ (黄色) ──┐  ┌─── APP (緑) ───────────┐
  │  [lb-01]        │  │  [app-01]   [app-02]    │
  └─────────────────┘  └────────────────────────┘

  ★ ポイント: エリアを使うと
    「このサーバがどのネットワークに属するか」が
    見た目ですぐわかるようになります。


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 07                                                    ║
║  エリアを追加・使ってみよう                                  ║
╚══════════════════════════════════════════════════════════════╝

  ▼ エリアの追加方法

    1. 左パネルの「エリア追加」セクションを見つける
    2. [MGMT] [DMZ] [APP] などのボタンをクリック
       → キャンバス中央に四角形のエリアが追加されます
    3. 独自の名前のエリアが欲しい場合は
       テキストボックスに名前を入力して [+] をクリック

  ▼ エリアにノードを入れるには

    1. エリアをドラッグして移動させ、ノードを囲む位置に置く
       (または先にエリアを置いてからノードをその上に移動する)
    2. ノードが視覚的にエリアの中に収まればOKです

    ※ エリアを移動すると中のノードも一緒に動きます

  ▼ エリアのサイズ変更

    エリアの右下にある小さなハンドルをドラッグすると
    サイズを変えられます

  ▼ エリア名の変更

    エリア名の文字をダブルクリックすると編集できます


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 08                                                    ║
║  「接続線」って何？                                          ║
╚══════════════════════════════════════════════════════════════╝

  ノードとノードの間を結ぶ矢印が「接続線」です。
  「どこからどこへ、どんな通信が流れているか」を表します。

  このアプリの接続線には3種類のスタイルがあります。

  ────────────────────────  青・実線   = 業務通信
                              (ユーザー向けの通常の通信)
                              例: HTTP, HTTPS, PostgreSQL

  - - - - - - - - - - - - -  緑・破線   = 管理通信
                              (サーバを操作・管理する通信)
                              例: SSH

  .............. ............  紫・点線   = 監視通信
                              (サーバの状態を見張る通信)
                              例: SNMP, ICMP

  線の上には「SSH/22」や「HTTP/80」のようなラベルも表示できます。
  (22 や 80 は「ポート番号」といって、通信の種類を区別する番号です)


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 09                                                    ║
║  ノードをつないでみよう                                      ║
╚══════════════════════════════════════════════════════════════╝

  ▼ 接続線の作り方

    STEP 1: ツールバーの [Select OFF] ボタンをクリック
            → [Select ON] に切り替わります

    STEP 2: つなぎたい「ノードAをクリック」
            → ノードの右上に番号 "1" が表示されます

    STEP 3: つなぎたい「ノードBをクリック」
            → ノードの右上に番号 "2" が表示されます

    STEP 4: ツールバーの [Link →] をクリック
            → AからBへの矢印(接続線)が引かれます
            → [Link ↔] なら双方向の矢印になります

    STEP 5: もう一度 [Select ON] をクリックして選択モードを終了

  ▼ 接続線の通信種別・プロトコルを設定するには

    1. 接続線をクリックする
    2. 右パネルに「接続線詳細」が表示されます
    3. 通信種別(業務/管理/監視)を選ぶ
    4. プロトコル(SSH/HTTP等)を選んで [+] で追加する

  ▼ 接続線を削除するには

    接続線をクリック → 右パネルの [この接続線を削除]


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 10                                                    ║
║  キャンバスを自由に動かそう                                  ║
╚══════════════════════════════════════════════════════════════╝

  構成図が大きくなってきたら、キャンバスを動かして
  全体を確認しましょう。

  ▼ ズームイン・アウト
    マウスのホイールを回す
    → 上に回す = 拡大 / 下に回す = 縮小

  ▼ キャンバスを移動する (パン)
    ノードのない背景の部分をドラッグする
    または マウスの中ボタン(ホイールボタン)を押しながらドラッグ

  ▼ 元の表示に戻す
    ツールバーの [⌖ Home] ボタンをクリック
    → ズームが100%に戻り、図の中心が画面の中央に来ます

  ▼ グリッドスナップ
    ツールバーの [Grid ON/OFF] をクリック
    → ONにするとノードが格子状の点にぴったり揃って動きます
    → きれいに整列させたいときに便利です


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 11                                                    ║
║  間違えても大丈夫！元に戻せます                              ║
╚══════════════════════════════════════════════════════════════╝

  操作を間違えてしまっても、いつでも元に戻せます。

  ▼ 操作を1つ取り消す (Undo)
    キーボード: Ctrl + Z
    または: トップバーの [↩ Undo] ボタン

  ▼ 取り消しを元に戻す (Redo)
    キーボード: Ctrl + Y
    または: トップバーの [↪ Redo] ボタン

  ▼ 自動保存について
    このアプリは操作のたびに自動でデータを保存しています。
    アプリを閉じても、次に開いたときに続きから作業できます。

  ★ データのバックアップをとりたいときは
    トップバーの [Export JSON] から
    図全体をJSONファイルとして書き出せます。
    このファイルを [Import JSON] で読み込めば
    いつでも復元できます。


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 12                                                    ║
║  実際に触ってみよう ― サンプル構成を読み解く                ║
╚══════════════════════════════════════════════════════════════╝

  アプリを起動すると最初から「三層Webシステム」の
  サンプル構成図が表示されています。

  登場するノードの意味はこうです。

    jump-01 (SV) : 管理者がサーバに入るときに
                   最初に踏む「踏み台サーバ」
                   → ここを経由しないと他のサーバに入れない

    lb-01   (LB) : ロードバランサ
                   → ユーザーからのアクセスをapp-01/app-02に振り分ける

    app-01  (SV) : アプリケーションサーバ1号機
    app-02  (SV) : アプリケーションサーバ2号機
                   → Webアプリの処理を担当

    db-01   (DB) : データベースサーバ
                   → ユーザーのデータを保存・管理

    mon-01  (MN) : 監視サーバ
                   → 全サーバの死活監視・メトリクス収集

  線の色と意味:
    青の実線  : HTTP通信 (業務通信)
    緑の破線  : SSH通信  (管理通信)
    紫の点線  : SNMP通信 (監視通信)

  ★ 各ノードをクリックして右パネルを見てみてください。
    IPアドレスや役割の情報が確認できます。


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 13                                                    ║
║  自分で構成図を作ってみよう ― 練習課題                      ║
╚══════════════════════════════════════════════════════════════╝

  サンプル構成をもとに、以下を試してみてください。

  【練習 1】ノードを追加してみる
    左パネルから [User/Client] を追加して
    名前を「User」に変更してみましょう。
    lb-01への接続線を引いて [業務通信] を設定してみましょう。

  【練習 2】エリアを見てみる
    MGMTエリア(水色)をドラッグして動かしてみましょう。
    jump-01とmon-01が一緒に動くことを確認しましょう。

  【練習 3】接続線を編集してみる
    lb-01→app-01の接続線をクリックして
    右パネルでプロトコルを確認・変更してみましょう。

  【練習 4】Export してみる
    トップバーの [Export JSON] をクリックして
    ファイルを保存してみましょう。
    保存したJSONファイルをテキストエディタで開いてみると
    データの構造が確認できます。


╔══════════════════════════════════════════════════════════════╗
║  SLIDE 14                                                    ║
║  まとめ                                                      ║
╚══════════════════════════════════════════════════════════════╝

  このアプリで学べること・できること

  ✔ ネットワーク構成図の基本的な読み方・書き方
  ✔ サーバの種類と役割の違い (SV/LB/DB/MN...)
  ✔ セグメント(エリア)の概念と重要性
  ✔ 通信の種類の違い (業務/管理/監視)
  ✔ プロトコルとポート番号の組み合わせ (SSH/22, HTTP/80...)
  ✔ 三層Webシステムの基本構成

  ★ 上手に使うためのポイント

    1. まずエリア(MGMT/DMZ/APP/DB)を先に配置する
    2. 次にノードをエリアの中に入れる
    3. 最後に接続線を引いて通信種別を設定する

    この順番で描くと整理しやすいです。

  ★ 困ったときは
    ・[⌖ Home] でいつでも全体表示に戻れます
    ・Ctrl+Z でいつでも操作を取り消せます
    ・[Export JSON] で定期的にバックアップを保存しましょう


╔══════════════════════════════════════════════════════════════╗
║  補足: よく使うキーワード集                                  ║
╚══════════════════════════════════════════════════════════════╝

  ノード        : 構成図上のコンピュータや機器1つ
  エッジ/接続線 : ノードとノードをつなぐ矢印
  セグメント    : 同じネットワークに属する機器のグループ
  IPアドレス    : ネットワーク上の住所 (例: 192.168.1.1)
  ポート番号    : 通信の種類を区別する番号 (例: 22=SSH, 80=HTTP)
  SSH           : サーバをリモート操作するための通信
  HTTP/HTTPS    : Webページを見るときの通信
  SNMP          : ネットワーク機器の状態を監視する通信
  ロードバランサ: 複数サーバへアクセスを分散させる機器
  踏み台サーバ  : 管理操作の入口となるサーバ (セキュリティのため)
  DMZ           : 外部インターネットに公開する区域
  MGMT          : Management の略。管理用ネットワーク
